1️⃣ Product Overview

Professionals in architecture, carpentry, interior design, and CNC machining often measure objects/spaces using AR tools, Unity scenes, or other systems. These tools produce raw 3D coordinates, but converting that data into a DXF CAD file requires expertise and expensive software.

This MVP provides a simple web-based tool where users upload raw 3D coordinate data, and the system converts it to a clean, accurate, AutoCAD-compatible DXF file, downloadable instantly.

This is the core engine of your future AR → DXF mobile app.

2️⃣ MVP Goal
🎯 Primary Objective

Build a browser-only tool (no backend) that:

Accepts raw 3D point data (x, y, z).

Converts points to a clean 2D geometry.

Generates an accurate DXF file.

Lets the user download the file.

This validates the hardest part of the final product:
➡️ The geometric transformation
➡️ The DXF export pipeline

4️⃣ User Stories
MVP User Stories (Core)

As a user, I want to upload a file containing raw 3D points so I can convert them into CAD-friendly data.

As a user, I want the system to automatically convert 3D points into clean 2D geometry.

As a user, I want to download a DXF file generated from the cleaned points.

As a user, I want the DXF file to open correctly in AutoCAD, ArtCAM, or Fusion360.

Edge Cases

If raw data is incomplete, I want the tool to show a validation error.

If the format is wrong, I want the tool to show the expected format.

If points are noisy, I want the converter to smooth & round automatically.

6️⃣ Input Format Requirement
Accepted Input (for MVP):
[
  { "x": 1.245, "y": 0.002, "z": -0.532 },
  { "x": 3.812, "y": 0.004, "z": -0.519 },
  { "x": 3.810, "y": 0.003, "z": 1.436 }
]


Values in meters

Raw AR world coordinates

y axis is height (ignored for floor)

7️⃣ Core Functional Requirements
A. Raw Data Parsing

Validate JSON structure

Check presence of x, y, z fields

Minimum 2 points required

Detect broken or empty arrays

B. Geometric Transformation

Convert 3D → 2D using (x, z)

Convert meters → mm

Remove decimals (round to nearest mm)

Normalize: shift all points so first point = (0,0)

Store points in order of input

C. DXF Generation

Generate LWPOLYLINE entity

Add vertices in order

Close polyline (optional toggle)

Allow download as output.dxf

D. UI

Simple page

Drag & drop or file input

Display cleaned 2D points (debug table)

Display success/failure messages

Export DXF button

8️⃣ Non-Functional Requirements
Requirement	Target
Accuracy	< 1 mm rounding error
Compatibility	AutoCAD R14+ opens file
Browser Support	Chrome, Safari, Firefox
Performance	Convert < 200ms for 100 points
Security	All local; no data sent to server
Accessibility	Simple, no dependency on screen readers
